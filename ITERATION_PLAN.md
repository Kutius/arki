## Phase 4: Windows 集成 + 体验打磨

### 4.1 Windows Shell 集成

**目标**：在 Windows 资源管理器中右键即可操作归档文件。

| 功能 | 说明 |
|------|------|
| 右键菜单 | "Extract to..." / "Create archive" |
| 文件关联 | 注册为 .zip/.7z/.rar 等格式的默认打开方式 |
| 图标覆盖 | 在资源管理器中显示归档文件的自定义图标 |

**实现方式**：
- Tauri v2 支持 `tauri-plugin-shell` 和自定义安装器
- NSIS 安装器支持注册 Shell Extension
- 需要 COM 组件注册（Windows 特定）

**风险**：Shell Extension 需要管理员权限注册，可能触发杀毒软件误报。

### 4.2 自动更新

**目标**：用户无需手动下载新版本。

**实现方式**：
- Tauri 内置 `tauri-plugin-updater`
- 需要一个更新服务器（GitHub Releases 或自建）
- 支持静默检查 + 提示更新

## 技术决策

### 1. Shell Extension 方案选择

**当前实现：SystemFileAssociations 注册表方式**
- ✅ 已实现，Win10/Win11 经典菜单（"显示更多选项"）均可用
- ✅ 无需管理员权限（HKCU）
- ❌ Win11 默认右键菜单不显示，需点击"显示更多选项"

**Win11 现代菜单方案：Sparse Package + COM DLL**
- 需要实现 IExplorerCommand COM 接口的 DLL
- 需要 AppxManifest.xml 声明上下文菜单
- 安装时注册 Sparse Package
- 这是 7-Zip 22.00+ 和 WinRAR 6.11+ 使用的方案
- 预估工作量：3-5 天

**决策**：当前使用 SystemFileAssociations 方案（已完成）。Win11 现代菜单作为后续迭代目标。

### 2. 更新服务器方案

**选项 A：GitHub Releases（推荐）**
- 优点：免费，稳定，与代码仓库集成
- 缺点：需要 GitHub 账号

**选项 B：自建服务器**
- 优点：完全控制
- 缺点：需要维护服务器

**决策**：选择 GitHub Releases，因为项目已经在 GitHub 上。

### 3. 流式解压方案

**选项 A：tokio 异步运行时**
- 优点：性能最好，Rust 生态标准
- 缺点：增加复杂度

**选项 B：std::thread 简单线程**
- 优点：简单，易于理解
- 缺点：性能不如 tokio

**决策**：选择 std::thread，因为当前需求不需要高并发，保持简单。

## 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Shell Extension 触发杀毒软件 | 用户无法安装 | 提供便携版，添加数字签名 |
| 自动更新服务器不稳定 | 用户无法更新 | 提供手动更新选项 |
| 大文件解压内存溢出 | 应用崩溃 | 流式解压，内存限制 |
| Windows 版本兼容性 | 部分功能不可用 | 测试 Windows 10/11 |
